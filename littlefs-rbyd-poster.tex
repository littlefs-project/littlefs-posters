To support increasingly dense storage, we are trying to find a B-tree with the
following constraints: (1)~strictly bounded-RAM, i.e. no non-tail
recursion, (2)~RAM usage independent of block size, and (3)~incremental,
i.e. does not rely on full block rewrites.

These constraints rule out existing B-tree designs.
Traditional B-trees implement inner nodes using dense arrays, and while
this may be optimal in terms of branching-factor and storage usage,
updates require rewriting full blocks.
Log-backed B-trees, such as those found in bcachefs~\cite{bcachefs},
support incremental updates, but rely on reconstructing logs in RAM.

Previous versions of \lfs+ avoided B-trees for this reason~\cite{TODO, TODO},

\section{Design}
\label{sec:design}
%-------------------------------------------------------------------------------

The core idea is to replace the inner nodes of traditional
B-trees~\cite{btree} with \emph{red-black-yellow Dhara (rbyd)} trees---a
self-balancing order-statistic variant of the Dhara tree implemented in the
Dhara FTL~\cite{dhara}.
The result is a flexible B-tree, with inner nodes that can be
incrementally manipulated without reading into RAM.

tree when keys are sparse.
